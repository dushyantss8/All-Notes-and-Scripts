# PHP Anonymous Classes: -

## 1. Introduction to Anonymous Classes

Anonymous classes in PHP allow you to create class instances **without defining a named class**. They are useful when you need one-off objects with custom behavior.

### Basic Syntax

```php
$object = new class {
    // class body
};
```

This creates an object whose class name is dynamically generated by PHP.

### Example: Creating an Anonymous Class Instance

```php
$object = new class {};
var_dump($object);
```

---

## 2. Constructors and Property Promotion

Anonymous classes can define constructors and accept parameters just like regular classes.

### Example: Passing Constructor Arguments

```php
$object = new class(1, 2, 3) {
    public function __construct(
        public $x,
        public $y,
        public $z
    ) {}
};

var_dump($object);
```

Result:

```
object(class@anonymous) {
    ["x"]=>1
    ["y"]=>2
    ["z"]=>3
}
```

---

## 3. Inheritance, Interfaces, and Traits

Anonymous classes support all major OOP features:

* Extending other classes
* Implementing interfaces
* Using traits

### Example

```php
$object = new class extends SomeClass implements SomeInterface {
    use SomeTrait;

    public function someMethod() {
        // implement interface method
    }
};
```

---

## 4. Class Name Resolution

Although anonymous classes do not have explicit names, the engine generates internal names.

```php
echo get_class($object);
```

However, because these names are unpredictable, developers should **treat anonymous classes as unnamed** and avoid relying on the generated name.

---

## 5. Type Hinting Limitations

You **cannot type-hint an anonymous class directly**.

### Invalid

```php
function foo(class@anonymous $obj) {} // not possible
```

### Valid Approach: Type-hint an Interface

```php
interface MyInterface {
    public function test();
}

$object = new class implements MyInterface {
    public function test() {}
};

function foo(MyInterface $obj) {
    var_dump($obj);
}

foo($object);
```

---

## 6. Using Anonymous Classes Inside Other Classes

You may return anonymous class instances from class methods.

### Example

```php
class A {
    public function bar(): object {
        return new class {};
    }
}

$a = new A();
var_dump($a->bar());
```

Return type must be a general `object` because anonymous classes cannot be named.

---

## 7. Accessing Outer Class Properties

Inside an anonymous class defined within another class, `$this` refers **only to the anonymous class**, not the outer class. Thus, you cannot directly access properties or methods of the parent class:

### Not Allowed

```php
new class {
    public function example() {
        echo $this->x; // fails
    }
};
```

### Solutions

#### A. Use inheritance

```php
class A {
    public function foo() { return 'foo'; }
}

$object = new class(1, 2) extends A {
    public function __construct(public $x, public $y) {
        parent::__construct();
    }
};
```

#### B. Pass values explicitly

```php
class A {
    public $x = 1;
}

$a = new A();

$object = new class($a->x) {
    public function __construct(public $x) {}
};
```

#### C. Pass the entire object

```php
$object = new class($a) {
    public function __construct(public $parent) {}

    public function example() {
        return $this->parent->foo();
    }
};
```

## 8. Use Cases for Anonymous Classes

According to the PHP RFC that introduced the feature, primary use cases include:

### 1. Mocking in Testing

Anonymous classes simplify the creation of mock objects for test scenarios.

### 2. Restricting Class Usage to Local Scope

Helps avoid polluting the global namespace with small, one-off classes.

### 3. Micro-Optimization

Avoids autoloader calls for very simple classes, offering minor performance benefits.

# Summary of Key Concepts

| Feature                 | Supported by Anonymous Classes? | Notes                                                    |
| ----------------------- | ------------------------------- | -------------------------------------------------------- |
| Constructors            | Yes                             | Can take arguments                                       |
| Property Promotion      | Yes                             | Fully supported                                          |
| Inheritance             | Yes                             | Can `extend` a class                                     |
| Interfaces              | Yes                             | Enables type hinting via interfaces                      |
| Traits                  | Yes                             | Same as regular classes                                  |
| Access to outer `$this` | No                              | Must use inheritance or pass values explicitly           |
| Type Hinting            | Limited                         | Cannot type-hint class name, only interfaces or `object` |
